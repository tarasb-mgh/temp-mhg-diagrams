# Release v2026.02.23

**Date**: 2026-02-23
**Previous release**: v2026.02.11
**Scope**: 6 repositories, 157 commits

## What's New (User-Facing)

- **Google sign-in** — users can now log in with their Google account on both the chat and workbench applications, removing the need for email/OTP-only authentication.
- **Responsive mobile layout** — both chat and workbench apps now adapt to phone and tablet screens with mobile-first layouts and are installable as Progressive Web Apps (PWAs).
- **Dedicated workbench application** — reviewers and supervisors now access a separate workbench at its own domain, optimised for review workflows with a collapsible sidebar and dedicated navigation.
- **Supervisor role** — new supervisor review interface with grade description tooltips, group-level user creation, and supervision dashboard.
- **Review queue tagging** — reviewers can tag chat sessions and filter the review queue by tags for faster triage.
- **Language improvements** — missing translations added for review features across English, Ukrainian, and Russian.
- **OTP toggle** — OTP verification can be disabled per-space when Google OAuth is the primary authentication method.

## Technical Changes

- Frontend monorepo split into `chat-frontend`, `workbench-frontend`, and `chat-frontend-common` (shared auth, API client, permissions).
- Backend surface split with `SERVICE_SURFACE` environment variable routing chat vs workbench API surfaces.
- Refresh token storage migrated from PostgreSQL to Redis (GCP Memorystore) with VPC connector.
- Cross-surface authentication hardening with secure, partitioned refresh-token cookies.
- Infrastructure provisioning: workbench domain (DNS records, SSL certificates, HTTPS load balancer, Cloud Run service), Redis Memorystore instance, VPC connector.
- 10 new database migrations (013–022) covering review system, supervision, tagging, crisis detection markers, and Google OAuth settings.
- End-to-end test suite covering authentication flows, review system, tagging, responsive layouts, cross-surface routing, and Google OAuth.
- Per-application CI/CD deploy workflows with manual rollback support.
- GCP secret pre-flight validation added to deploy pipeline.
- Google OAuth client ID injected via Cloud Run secrets.
