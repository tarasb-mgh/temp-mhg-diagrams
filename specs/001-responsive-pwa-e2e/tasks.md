## Dependencies & Execution Order

### Phase Dependencies

- **Phase 1 (Setup)**: Starts immediately.
- **Phase 2 (Foundational)**: Depends on Phase 1; blocks all user stories.
- **Phase 3 (US1)**: Depends on Phase 2; MVP-first target.
- **Phase 4 (US2)**: Depends on Phase 2; can run in parallel with US1 once foundations are stable.
- **Phase 5 (US3)**: Depends on Phases 3 and 4 (tests validate responsive + PWA behavior).
- **Phase 6 (Polish)**: Depends on completion of all user stories.

### User Story Dependencies

- **US1 (P1)**: No dependency on other stories after foundational tasks. Can start as soon as Phase 2 completes.
- **US2 (P2)**: No dependency on US1 functionally (PWA setup is independent), but practically benefits from responsive layouts being in place. Can run in parallel.
- **US3 (P3)**: Depends on US1 and US2 — tests validate the behavior those stories implement. Must run after US1/US2 are at least partially complete.

### Within Each User Story

- Chat app and workbench app tasks within the same story can run in parallel (different repos).
- Shell/navigation changes should precede page-level changes within each app.
- PWA hook → component → wiring order within US2.

### Parallel Opportunities

- Setup tasks `T002`–`T004` (different repos).
- Foundational tasks `T005`–`T008` (different repos/files).
- US1: Chat tasks `T009`–`T014` vs workbench tasks `T015`–`T020` (different repos).
- US2: Chat tasks `T022`–`T024` vs workbench tasks `T025`–`T027` (different repos).
- US3: Chat responsive `T030` vs workbench responsive `T031` (different test files).
- US3: Chat PWA `T033` vs workbench PWA `T034` (different test files).
- Polish: Screenshot/documentation tasks `T038`–`T043`.

## Implementation Strategy

### MVP First (User Story 1 Only)

1. Complete Phases 1–2 (setup + foundation).
2. Complete Phase 3 (US1 — responsive layouts).
3. Validate both apps at phone/tablet/desktop viewports.
4. Demo/deploy responsive MVP increment.

### Incremental Delivery

1. Foundation (Phases 1–2).
2. US1 (responsive layouts) → validate independently → deploy.
3. US2 (PWA installability) → validate independently → deploy.
4. US3 (E2E responsive tests) → validate CI gate → deploy.
5. Polish + release evidence + documentation.

### Parallel Team Strategy

1. Team completes Setup + Foundational together.
2. Split by repo:
   - Dev A: Chat app (US1 chat tasks → US2 chat tasks)
   - Dev B: Workbench app (US1 workbench tasks → US2 workbench tasks)
   - Dev C: E2E tests (US3, after US1/US2 in progress)
3. Converge for Phase 6 evidence, documentation, and PR/release hygiene.
