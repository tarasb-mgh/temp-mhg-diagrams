# Implementation Plan: Workbench Sidebar Menu Reorganization

**Branch**: `033-workbench-menu-groups` | **Date**: 2026-03-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/033-workbench-menu-groups/spec.md`

## Summary

Reorganize the workbench sidebar from a flat 15-item list with ad-hoc grouping into a structured, data-driven menu with 3 collapsible groups (Reviews, Surveys, People & Access), consistent naming, unique icons, and 4 newly surfaced review sub-routes. The implementation is frontend-only, targeting `workbench-frontend` with changes to navigation config, Zustand state, localization files, and sidebar rendering. Based on 029-workbench-ux-wayfinding findings that identified flat menu structure, duplicate icons, and missing routes as primary navigation confusion sources.

## Implementation Approach

### Phase 1: Localization (i18n keys)

Add all new and renamed localization keys to `en.json`, `uk.json`, `ru.json` before any component changes. This allows incremental development ‚Äî components can reference new keys immediately.

Changes per locale file:
- 3 new group heading keys (`group.reviews`, `group.surveys`, `group.peopleAccess`)
- 8 new/renamed item keys (reviewQueue, reviewDashboard, teamDashboard, escalations, reviewTags, reviewSettings, surveyTemplates, testerTags)
- 5 shortened existing labels (users, groups, privacy, reviewReports ‚Üí reports, tagManagement ‚Üí reviewTags)

### Phase 2: State Management (Zustand)

Add to `workbenchStore.ts`:
- `navGroupCollapsed: Record<string, boolean>` state field
- `toggleNavGroup(groupId: string)` action
- Extend `partialize` to include `navGroupCollapsed` for localStorage persistence

### Phase 3: Navigation Config Restructure

In `WorkbenchLayout.tsx`:
1. Define `NavGroupConfig` interface
2. Replace flat `navItems` array with structured config:
   - `dashboardItem`: single `NavItemConfig` (ungrouped, top)
   - `navGroups`: `NavGroupConfig[]` array with reviews, surveys, peopleAccess groups
   - `settingsItem`: single `NavItemConfig` (ungrouped, bottom)
3. Update all icon imports (add new icons, remove unused)
4. Update all `labelKey` references to new key names

### Phase 4: Sidebar Rendering

In `WorkbenchLayout.tsx`:
1. Replace `mainNavItems` / `researchNavItems` split with group-based rendering
2. Add group heading component with collapse toggle (chevron indicator)
3. Add auto-expand logic: if current path matches any item in a collapsed group, force expand
4. Maintain Group Resources block position (after groups, before Settings)
5. Maintain `allNavItems` derivation for breadcrumb active-state calculation

### Phase 5: Validation

1. Visual verification across all 5 roles
2. Collapse state persistence across navigation
3. Empty group hiding verification
4. Mobile responsive verification
5. Breadcrumb correctness for newly surfaced routes
